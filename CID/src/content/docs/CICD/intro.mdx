CI/CD står for Continuous Integration/Continuous Deployment, og er en praksis som brukes for å automatisere bygging, testing og utrulling av programvare.

CI/CD er er sentralt i DevOps, som er en tilnærming innen programvareutvikling som fremmer samarbeid mellom utviklere og driftspersonell.

Ved å bruke CI/CD kan teamet ditt raskt og effektivt levere programvare av høy kvalitet til brukerne.
Dette kan bidra til å redusere feil, øke produktiviteten og forbedre brukeropplevelsen.

På denne nettsiden vil du lære om de grunnleggende prinsippene bak CI/CD og hvordan disse kan anvendes i arbeidslivet.
Vi vil demonstrere hvordan CI/CD fungerer ved hjelp av verktøy som Jenkins og Docker.
I tillegg vil vi ta for oss beste praksiser, samt vanlige utfordringer som kan oppstå under implementeringen.
